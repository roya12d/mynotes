state

وقتی state یک کلاس تغییر می کند آن کامپوننت دوباره رندر می شود و تمام فرزندان ان کامپوننت هم دوباره رندر می شوند زیرا فرزند یک کامپوننت چیست جز فراخوانی کامپونت دیگر درون متد رندر ان کامپوننت پس بدییهی است که دوباره فراخوانی شود

استیون می گوید اگر دیدی کامپوننت نیاز به اپدیت شدن دارد فکر استفاده از state ها باش

down data flow

down data flow یک مفهوم در ری اکت است که می گوید بالاترین کامپوننت باید به داده ها و کار با api ها و fetch کردن داده ها دسترسی و مسئول باشد و فرزندان از این داده های گرفته شده استفاده می کنند

تبدیل function base component به class base component

هنگام تبدیلfunction base component باید حواسمان باشد که شیوه ی ارجاع به props در این دو فرق دارد از function ase component ما به props ها از طریق نام ارگومانی که خودمان در تابع مشخص می کنیم دسترسی داریم مثلا props.name اما در کلاس ها ما از طریق this.props دسترسی داریم که اینجا دیگر props یک کلمه ی رزرو شده است پس در صورت این که قصد انجام چنین کاری را داشتیم باید مراقب این تغییرات هم باشیم

رندر کردن یک لیست از عناصر

استیون وقتی دارد سایت شبیه یوتیوب را می سازد به این ترتیب اینکار را انجام می دهد که یک کامپونتت مسئول ایجاد تگ بزرگتر یعنی ul می شود و یک کامپوننت مسئول ایجاد تگ کوچکتر یعنی li می شود داده را برای پدر می فرستیم پدر روی ان داده حلقه ی map می زند و به تعدا عناصر موجود در ان داده کامپوننت فرزندش را قراخوانی می کند و در این فراخوانی ها البته داده هایی که نیاز است را به عنوان props برایش ارسال می کند. نکته ی دیگری که باید در فراخوانی کامپوننت های ایجاد کننده زیرلیست ها رعایت کنیم این است که هر المان باید حاوی یک prop به نام key باشد تا react بتواند یونیک بودن ان را تشخیص دهد احتمالا برای ما که جنگو کار می کنیم این key همان pk می شود نمونه:

call back

call back در واقع function است که ما از والد برای فرزند یا نوه اش می فرستیم. کاری که این تابع می تواند انجام دهد این است که state پدر را تغییر دهد.
